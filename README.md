# ❄️ Modular NixOS & Home Manager Configuration

Welcome to my personal, flake-based NixOS configuration. This setup is fully modularized, keeping system-level configurations and user-level dotfiles (via Home Manager) cleanly separated and organized.

> 💡 **Note:** I am a Computer Science student transitioning to NixOS. I used an AI assistant to help me structure parts of these configurations, handle the syntax, and write clean, informative comments within the codebase.

---

## 📂 Repository Structure

The project follows a clean, modular Nix flake directory layout:

```text
nixos-config
├── flake.nix                  # Main entry point and inputs resolution for the system configuration
├── flake.lock                 # Lockfile pinning precise upstream versions of all inputs
├── README.md                  # System architecture and documentation file
│
├── hosts/
│   └── default/
│       ├── configuration.nix  # System-level settings, hardware drivers, and core configuration
│       └── hardware-configuration.nix # Autogenerated hardware device mapping and kernel modules
│
└── home/
    ├── colorScheme.nix        # Centralized theme specification to easily swap color schemes globally
    ├── ziad.nix               # Main user profile configuration importing modular packages
    └── modules/               # Modular standalone configurations for apps & environment
        ├── ghostty.nix        # Ghostty terminal emulator styling & configurations
        ├── hyprland.nix       # Hyprland window manager module declaring standalone Lua configurations
        ├── neovim.nix         # Neovim text editor setup, LSP configurations, and plugins
        ├── shell.nix          # Shell configurations, CLI tools, aliases, and environment variables
        └── waybar.nix         # Waybar status bar customization and styling
